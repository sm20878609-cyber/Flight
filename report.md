# 期中專題報告：AI 智慧航班機票搜尋與推薦系統

## 1. 專案名稱
AI 智慧航班機票搜尋與推薦系統 (AI-Powered Flight Ticket Search and Recommendation System)

## 2. 選擇題目
**Topic 3: AI-Powered True Cost Flight Comparison System** (延伸與改版)

## 3. 系統目標
本系統旨在協助使用者在海量航班中，快速找出最符合個人偏好的機票。系統能查詢即時航班機票，並根據「機票價格」、「飛行時間」、「轉機次數」等關鍵特徵，結合使用者設定的偏好（例如：最便宜、最快抵達），運用機器學習演算法 (KNN) 進行 AI 推薦排序，並透過 KMeans 自動將航班分群，提供視覺化的分析結果。

## 4. 資料來源
本系統支援多重資料來源，具備高度容錯的備援機制：
- **Google Flights scraper**：主要資料來源，透過開源套件 `fast-flights` 動態爬取即時航班。
- **cached_scraped_flights.csv**：爬蟲成功後自動儲存的快取檔，於爬蟲被阻擋時做為第一順位備援。
- **sample_flights.csv**：系統內建的靜態資料，於完全無網路或快取時的最終備援。
- **使用者上傳 CSV**：允許使用者匯入自定義格式的機票資料進行 AI 運算。

## 5. 資料處理
從爬蟲取得的非結構化字串中進行以下處理：
- **欄位標準化**：統一整理出 `flight_id`, `airline`, `price`, `duration_minutes`, `stops` 等 15 項特徵。
- **缺失值處理**：自動剔除票價為 0 或飛行時間異常的資料。
- **時間特徵轉換**：將字串格式的 `4 hr 15 min` 轉換為純數值 `255` 分鐘；並從 `departure_time` 中萃取出起飛小時 (`departure_hour`)。
- **重複航班移除**：比對航空公司、起降時間與價格，去除重複抓取的冗餘資料。

## 6. AI/ML 方法
本系統採用了兩種機器學習方法來提升決策品質：
1. **KNN-based Recommendation (推薦排序)**：
   - 使用特徵：`price`, `duration_minutes`, `stops`, `departure_hour`
   - 使用 `StandardScaler` 將特徵標準化。
   - 根據使用者選擇的偏好，建構出理想點 (Ideal Vector)，並賦予不同特徵權重（例如選「最便宜」時，`price` 的權重最高）。
   - 計算各航班到 Ideal Vector 的歐氏距離，轉換成 0~100 的推薦分數，最後依分數高低排序。
2. **KMeans Flight Clustering (航班分群)**：
   - 使用特徵：`price`, `duration_minutes`, `stops`
   - 使用 KMeans 演算法自動將當次搜尋結果分為 2~4 群。
   - 系統依據群集內的平均票價與飛行時間，自動標註「低價航班」、「快速航班」或「高價航班」等易讀標籤。

## 7. GUI 操作流程
1. 於頂部過濾卡片選擇出發地與目的地。
2. 選擇「查詢日期」與「AI 推薦優先順序」。
3. 點擊「🔍 查詢並分析航班」，系統將即刻觸發爬蟲與 ML 運算。
4. 觀看上方的指標卡片與最推薦首選。
5. 於下方的五個頁籤中，分別瀏覽詳細的推薦優缺點、完整數據表、分群散佈圖、資料視覺化圖表，或直接下載分析結果 CSV。

## 8. 結果解釋
系統不僅給出排名，還透過簡單易懂的繁體中文解釋「推薦原因」與「優缺點」。例如，當航班票價顯著低於平均值時，會將其列為優點；若轉機次數過多，則列為缺點。這讓使用者不僅知其然，更知其所以然。透過 PCA 降維畫出的分群散佈圖，也能讓使用者一眼看出不同航空公司的定價策略區間。

## 9. 系統限制
- 爬蟲資料可能受來源網站變動影響，高頻率查詢可能導致 IP 暫時被鎖。
- 票價可能即時變動，實際價格以航空公司或訂票平台最後顯示為準。
- 本系統只做航班機票比較，不處理最終訂票付款。
